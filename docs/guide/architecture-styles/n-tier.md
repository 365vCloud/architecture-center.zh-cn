<span data-ttu-id="dc8c4-105">层是分离职责和管理依赖关系的方式。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-105">Layers are a way to separate responsibilities and manage dependencies.</span></span> <span data-ttu-id="dc8c4-106">每个层都有特定的责任。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-106">Each layer has a specific responsibility.</span></span> <span data-ttu-id="dc8c4-107">较高层可使用较低层中的服务，反之则不行。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-107">A higher layer can use services in a lower layer, but not the other way around.</span></span> 

<span data-ttu-id="dc8c4-108">层在物理上是分隔开的，在不同的计算机上运行。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-108">Tiers are physically separated, running on separate machines.</span></span> <span data-ttu-id="dc8c4-109">一个层可直接调用另一个层，或使用异步消息传递（消息队列）。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-109">A tier can call to another tier directly, or use asynchronous messaging (message queue).</span></span> <span data-ttu-id="dc8c4-110">虽然每个层可能托管在自己的层中，但这并不是必需的。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-110">Although each layer might be hosted in its own tier, that's not required.</span></span> <span data-ttu-id="dc8c4-111">多个层可能托管在同一层上。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-111">Several layers might be hosted on the same tier.</span></span> <span data-ttu-id="dc8c4-112">在物理上分隔层可以提高可伸缩性和复原能力，但因额外的网络通信也增加了延迟。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-112">Physically separating the tiers improves scalability and resiliency, but also adds latency from the additional network communication.</span></span>

<span data-ttu-id="dc8c4-117">N 层应用程序可以有封闭的层体系结构或开放的层体系结构：</span><span class="sxs-lookup"><span data-stu-id="dc8c4-117">An N-tier application can have a **closed layer architecture** or an **open layer architecture**:</span></span>

- <span data-ttu-id="dc8c4-118">在封闭的层体系结构中，层只能调用紧邻的下一层。</span><span class="sxs-lookup"><span data-stu-id="dc8c4-118">In a closed layer architecture, a layer can only call the next layer immediately down.</span></span>
