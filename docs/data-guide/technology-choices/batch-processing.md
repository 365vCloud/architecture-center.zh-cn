# <a name="choosing-a-batch-processing-technology-in-azure"></a><span data-ttu-id="0f34a-102">在 Azure 中选择批处理技术</span><span class="sxs-lookup"><span data-stu-id="0f34a-102">Choosing a batch processing technology in Azure</span></span>

<span data-ttu-id="0f34a-103">大数据解决方案经常使用长时间运行的批处理作业来筛选、聚合或者准备数据以用于分析。</span><span class="sxs-lookup"><span data-stu-id="0f34a-103">Big data solutions often use long-running batch jobs to filter, aggregate, and otherwise prepare the data for analysis.</span></span> <span data-ttu-id="0f34a-104">这些作业通常涉及从可缩放的存储（例如，HDFS、Azure Data Lake Store 和 Azure 存储）读取源文件，对源文件进行处理，并将输出写入到可缩放存储中的新文件。</span><span class="sxs-lookup"><span data-stu-id="0f34a-104">Usually these jobs involve reading source files from scalable storage (like HDFS, Azure Data Lake Store, and Azure Storage), processing them, and writing the output to new files in scalable storage.</span></span> 

<span data-ttu-id="0f34a-105">对这类批处理引擎的关键要求是需要它们能够扩展计算能力，以便处理大量数据。</span><span class="sxs-lookup"><span data-stu-id="0f34a-105">The key requirement of such batch processing engines is the ability to scale out computations, in order to handle a large volume of data.</span></span> <span data-ttu-id="0f34a-106">但是，不同于实时处理，批处理会有延迟（从数据引入到计算结果之间的时间），延迟范围为数分钟到数小时。</span><span class="sxs-lookup"><span data-stu-id="0f34a-106">Unlike real-time processing, however, batch processing is expected to have latencies (the time between data ingestion and computing a result) that measure in minutes to hours.</span></span>
